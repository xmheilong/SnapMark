## 3. 模式与状态机

### 3.1 模式关系

```
┌──────────────────────────────────────────────────┐
│                  主应用状态                       │
│                                                   │
│   ┌─────────────┐         ┌──────────────┐        │
│   │  标注模式    │  ←→    │  截图模式     │        │
│   │  (上游原) │ 互斥   │  (本功能新增) │        │
│   └─────────────┘         └──────────────┘        │
└──────────────────────────────────────────────────┘
```

**截图模式与标注模式互斥**：不能同时激活。

### 3.2 互斥规则

打开"截图模式"开关时：
- ✅ 暂停标注功能（快捷键不响应、工具栏按钮灰掉）
- ✅ **暂停鼠标点击特效**（避免干扰选区绘制视觉）⚠️ 详见 3.4
- ❌ **不动键盘回显**（不干扰截图）
- 退出截图模式时自动恢复以上状态

### 3.3 状态机

```
[标注模式]
   ↓ 用户打开截图开关
[截图模式 - 等待选区]
   ↓
   ├── Enter 键     → 截默认全屏 → 写剪贴板 → [截图模式 - 等待选区]
   ├── 鼠标拖动     → [截图模式 - 选区拖动中]
   │                    ↓ 松手
   │                  截选区 → 写剪贴板 → [截图模式 - 等待选区]
   ├── Esc 键       → [标注模式]
   └── 点 ✕ 按钮    → [标注模式]
```

**关键点**：截完一张后**留在截图模式**，等下一次操作（连续截多张是高频场景）。

### 3.4 鼠标点击特效的处理

虽然用户原话是"不动鼠标/键盘特效"，但鼠标点击特效在选区拖拽时会持续触发并产生视觉干扰。**建议在截图模式下也暂停鼠标点击特效**，原因：
- 选区拖拽过程会产生高频点击
- 视觉重叠影响截图选区绘制
- 退出截图模式自动恢复，对用户零负担

> 此项作为推荐实现，若用户坚持保留，标注清楚取舍即可。

## 4. 交互细节

### 4.1 进入截图模式

| 项 | 规格 |
|----|------|
| 触发 | 用户在主窗口设置页打开"截图模式"开关 |
| 视觉反馈 | 全屏覆盖层立即出现 |
| 默认选区 | **整个桌面（物理全屏）** |
| 提示文字 | 覆盖层底部居中：`拖动框选 · Enter 截全屏 · Esc 退出` |
| 退出入口 | 屏幕角落"✕ 退出截图模式"按钮 + Esc 键 |

### 4.2 全屏覆盖层视觉规格

```
┌─────────────────────────────────────────────┐
│                                             │
│        ┌─────────────────────────┐          │
│        │                         │          │
│        │     默认选区 = 全屏      │          │
│        │   (高亮边框 + 尺寸文字)   │          │
│        │                         │          │
│        └─────────────────────────┘          │
│                                             │
│      拖动框选 · Enter 截全屏 · Esc 退出       │
│                              [✕ 退出]        │
└─────────────────────────────────────────────┘
```

| 元素 | 规格 |
|------|------|
| 遮罩底色 | `rgba(0, 0, 0, 0.4)`（半透明黑） |
| 选区内 | 原画面正常显示（不被遮罩覆盖） |
| 选区边框 | 2px 高亮（如 `#4ecdc4`，与品牌色一致） |
| 尺寸文字 | 选区左上角或居中：`1920 × 1080` |
| 提示文字 | 覆盖层底部居中，白色，半透明背景 |
| 退出按钮 | 屏幕右上角小圆角按钮 |

### 4.3 选区操作

| 操作 | 行为 |
|------|------|
| 鼠标按下 + 拖动 | 重新画选区，松开后立刻截图 |
| 没拖动按 Enter | 截默认全屏 |
| 没拖动直接按 Esc | 退出截图模式 |
| 拖动中按 Esc | 取消当前选区，回到默认全屏（**不退出模式**） |
| 选区尺寸 < 10×10 | 视为无效，提示并保留当前状态 |

### 4.4 截图完成反馈

- **写剪贴板成功**：选区边框**闪烁一次**（200ms 内高亮 → 消失），无其他弹窗
- **不弹通知、不出声**——保持低干扰

## 5. 窗口状态管理

### 5.1 截图瞬间窗口状态

**决定：截图时保持窗口状态不变，不主动 hide/show 标注窗口。**

理由：
- 简化状态管理
- 上游标注窗口是 transparent 的，理论上不应被截到
- 用户明确要求"保持截图前状态一样"

### 5.2 已知风险与兜底

Windows GPU 合成（DWM）下，transparent 窗口**可能**留下隐约影子。处理：
- 不默认开启 hide/show
- **保留 `window.hide()` / `window.show()` 兜底代码**（约 5 行），作为开关函数
- 未来若用户反馈截到自己的窗口，5 分钟内可启用，无需重构

### 5.3 截图模式期间

- 标注窗口保持显示（用户在屏幕上能看到自己的标注按钮等 UI）
- 但标注功能本身**逻辑暂停**（按键不响应）

## 6. 错误处理

| 场景 | 行为 |
|------|------|
| 截图失败（罕见，如 DXGI 异常） | Toast 通知 + **自动退出截图模式** + 提示失败原因 |
| 剪贴板写入失败 | Toast 通知 + 退出截图模式（不 fallback 到文件，MVP 范围外） |
| 选区尺寸过小（< 10×10） | Toast 轻提示，**不退出模式** |
| 用户按 Esc 取消 | 正常退出，无提示 |

**Toast 形式**：屏幕底部居中浮层，1~2 秒自动消失，不打断用户操作。

## 7. 数据持久化

用现有依赖 `tauri-plugin-store`（已在 `package.json` 中）：

| Key | 类型 | 默认值 | 说明 |
|-----|------|--------|------|
| `screenshot_enabled` | boolean | `false` | 截图模式开关状态 |

**持久化策略**：应用重启后保持上次状态。
- 用户主动打开 → 下次启动仍然开启
- 用户主动关闭 → 下次启动关闭
- **默认关闭**，避免新用户首次启动被覆盖层吓到

## 8. 技术实现

### 8.1 技术栈选型

| 层 | 选型 | 理由 |
|----|------|------|
| 截图库（Rust） | `screenshots = "0.8"`（基于 xcap） | 跨平台一致的 API，Windows 上底层用 Win32 GDI / DXGI |
| 图像处理 | Rust `image` crate | tauri 已启 `image-png` feature，可直接编 PNG |
| 剪贴板写入 | 前端 `navigator.clipboard.write()` | 不需要 Rust 写 `arboard` crate，更轻 |
| 状态存储 | `tauri-plugin-store` | 项目已有依赖 |
| 覆盖层渲染 | React + Canvas2D | 性能足够 60fps，避免 WebGL 复杂度 |

### 8.2 Rust command 接口

```rust
#[tauri::command]
async fn capture_region(
    x: i32, y: i32, w: u32, h: u32
) -> Result<Vec<u8>, String>
// 返回 PNG 字节，前端再写剪贴板
```

错误用 `String` 返回（MVP 简化），后续可改为结构化错误类型。

### 8.3 前端调用流程

```
用户拖动选区 → 松手
    ↓
invoke('capture_region', { x, y, w, h })
    ↓ Rust 返回 PNG bytes
const blob = new Blob([pngBytes], { type: 'image/png' })
    ↓
navigator.clipboard.write([
    new ClipboardItem({ 'image/png': blob })
])
    ↓
选区闪烁动画（200ms）
```

## 12. 设计决策日志

按对话顺序记录的关键决策，便于追溯：

1. **项目定位**：fork → SnapMark，开源，不回 PR 上游
2. **命名规则**：应用名 SnapMark，identifier `io.github.xmheilong.snapmark`，仓库名沿用上游旧名
3. **范围收窄**：Windows only / 矩形 / 剪贴板 / 开关触发
4. **模式关系**：截图模式与标注模式互斥
5. **开关位置**：主窗口设置页（新建"截图"Tab）
6. **暂停范围**：只暂停标注，建议同时暂停鼠标点击特效，键盘回显不动
7. **截完行为**：留在截图模式（连击友好）
8. **窗口状态**：截图时不隐藏标注窗口（简化），保留 hide 兜底
9. **默认选区**：整个桌面（全屏）
10. **松手行为**：立刻写剪贴板 + 选区闪烁 200ms 反馈
11. **错误处理**：失败 Toast + 退出截图模式
12. **持久化**：screenshot_enabled 默认 false，重启保持
13. **剪贴板实现**：前端 Clipboard API，不走 Rust
14. **快捷键**：MVP 不做，留到后续
